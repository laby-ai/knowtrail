# 虚拟教室参考边界

本项目只把 OpenMAIC 作为内部产品与代码参考，不在用户界面、产物文案、按钮说明或默认导出中出现 OpenMAIC/MAIC 名称。

## 许可结论

- 根目录 `LICENSE` 与 `package.json` 标注为 MIT，商业使用通常允许。
- 仓库内存在不完全由根 MIT 覆盖的子包和资产：
  - `packages/mathml2omml` 为 `LGPL-3.0-or-later`。
  - `packages/@openmaic/renderer` 下有字体许可与归属要求。
  - `packages/pptxgenjs` 为第三方 MIT。
- 因此，安全策略是参考流程和数据结构，避免直接复制原 UI、品牌资源、字体资源、完整页面壳或子包实现。

## 可复用方式

- 可参考：课程大纲、场景确认、讲解/测验/项目任务、生成进度、产物导出等流程结构。
- 应重做：灵笔自己的前端壳、文案、图标、交互层级、预览页面、导出页面。
- 可作为过渡集成：本地 sidecar 的课堂运行时 iframe，但必须嵌在灵笔中间工作区内，由灵笔入口、返回、状态和最近课堂管理。
- 禁止默认暴露：原项目品牌名、原项目公网入口、原项目营销页、原项目字体包、原始 logo/banner。

## 当前实现约束

- 右侧 Studio 只作为入口和状态控制区，不承载完整课堂。
- 点击“虚拟教室”后，中间工作区直接打开本地 sidecar 课堂运行时。
- 中间工作区必须保留灵笔的返回、刷新、外部打开和失败恢复入口。
- 用户可见文本只使用“虚拟教室/课程大纲/课堂/测验/互动/导出”等产品语义。
- `/virtual-classroom` 不能作为外部参考项目的公开跳转页；它只服务灵笔工作台内的课堂体验。

## 发布前检查

- `pnpm run check:reference-license-risk` 必须通过，确认 MIT 根许可、LGPL/字体例外和边界文档仍被登记。
- `pnpm run check:reference-boundaries` 必须通过，确认用户可见源码没有参考项目名、历史项目名或旧服务名。
- 虚拟教室 smoke 必须验证课堂 iframe 无白屏、无 console/page error、无内部参考名泄漏。
