# Embedded Quiet Research Workbench Design

## 1. 背景与目标

KnowTrail 作为科教平台的嵌入式智能体时，当前存在三个连续的用户体验断点：

1. 从科教平台进入智能体时，首帧会短暂显示 KnowTrail 营销落地页，再切换到文献本列表，形成明显闪屏和路径噪声。
2. 嵌入态仍展示 KnowTrail 品牌和面向独立站的引导文案，与科教平台宿主品牌、返回路径和访客提示冲突。
3. 文献工作台三栏边界、卡片和空状态过于粗重，控件密度与层级不统一，不适合长时间科研阅读。

本设计选择已确认的 A 方向“Quiet Research”：以冷静的蓝灰中性色、细分隔线、紧凑层级和克制动效，形成安静、精细、可长时间使用的科研工作台。

## 2. 范围

### 2.1 本轮包含

- 科教平台嵌入态的首帧路由解析和落点。
- 嵌入态文献本首页与工作台的品牌隐藏规则。
- 文献本工作台顶部栏、三栏布局、空状态和右侧工具区的视觉密度。
- 嵌入态访客提示的尺寸与层级。
- hover、focus、面板折叠和页面进入等必要动效。
- 正式域名 1440px 用户路径、错误态、键盘焦点和无障碍回归。

### 2.2 本轮不包含

- 不重写文献问答、发现信源、PPT、科研图像或其他模型能力。
- 不改变任何 AI provider、计费、账号中心或高成本门禁策略。
- 不改变 guest workspace、account scope、持久化或数据隔离语义。
- 不重做手机布局；手机仅做必要的不回退检查。
- 不删除独立站的 KnowTrail 营销落地页或独立站品牌。

## 3. 入口与首帧状态

### 3.1 嵌入态

paper-host iframe URL 必须在首个可见渲染前解析出嵌入状态和目标视图：

- `view=notebooks`：直接显示文献本列表。
- `view=workbench` 且包含有效文献本上下文：直接显示工作台。
- 无有效目标时：回退到文献本列表，不回退到营销落地页。

嵌入态首个有意义画面不得出现以下内容：

- “开启科研模式”等营销 CTA。
- KnowTrail 品牌字样或独立站导航。
- 先显示落地页、再切换列表的闪屏。

实现应通过单一的初始视图解析模型确定首帧，而不是在客户端挂载后的 effect 中二次纠正已渲染页面。该模型只负责输入到视图状态的纯转换，页面仍拥有路由和宿主通信副作用。

### 3.2 独立站

独立访问根路径时继续保留营销落地页、KnowTrail 品牌和独立站导航。嵌入态优化不得改变独立站默认入口。

## 4. 品牌与宿主契约

- 嵌入态文献本首页和工作台不显示 KnowTrail 字样、品牌图标或“试用 KnowTrail”引导。
- 工作台左上仅保留：返回按钮、文献本标题、必要的轻量状态标记。
- 宿主仍拥有科教平台品牌、登录跳转和全局返回路径。
- paper-host 的 `postMessage` bridge、`hideVirtualClassroom`、workspace/account 隔离和退出契约保持不变。
- 独立站继续显示 KnowTrail 品牌，不使用全局字符串替换或删除品牌资源。

## 5. Quiet Research 视觉系统

### 5.1 色彩

| 语义 | 建议色值 | 用途 |
| --- | --- | --- |
| 页面背景 | `#F7F9FC` | 嵌入态工作区底色 |
| 主面板 | `#FFFFFF` | 左中右工作面 |
| 次级面 | `#FAFBFD` | 工具卡、空状态背景 |
| 分隔线 | `#E4E9F1` | 三栏和区块边界 |
| 主文字 | `#142033` | 标题和主要正文 |
| 次文字 | `#65738A` | 说明、状态、元数据 |
| 品牌动作 | `#2866D7` | 主按钮、选中、焦点 |
| 柔和蓝底 | `#EDF4FF` | 轻提示、选中背景 |
| 成功/警告/错误 | 沿用语义 token | 不用装饰色覆盖状态语义 |

不得用大面积高饱和渐变、强玻璃反射或多层发光占据工作区。现有 Liquid Glass 能力仅保留在宿主级或关键浮层，三栏主体改为安静的实体面与细边界。

### 5.2 字体与层级

- 工作台标题：18–20px，600 字重。
- 区域标题：14–16px，600 字重。
- 正文与控件：14px，400/500 字重。
- 元数据与辅助说明：12–13px。
- 行高保持 1.45–1.65，长文阅读区优先舒适度。
- 不用超大标题、全大写或纯装饰性渐变文字。

### 5.3 圆角、阴影与间距

- 主面板不再使用多层卡片包裹；栏间使用 1px 分隔线。
- 工具卡圆角 12–14px，边框 `#E4E9F1`，默认无明显阴影。
- hover 时仅提升边框/背景或出现极轻阴影，不产生明显位移。
- 基础间距采用 4/8/12/16/24px 节奏；同级工具之间优先 8–12px。

## 6. 三栏工作台

### 6.1 默认结构

- 左栏：约 272px，负责文献来源、上传和检索。
- 中栏：弹性宽度，负责文献问答与主要任务。
- 右栏：约 420px，负责产品工具、检索和生成结果。

窗口缩小时，优先压缩中栏并保留左右栏最低可用宽度；不得产生页面级横向滚动。折叠行为沿用现有状态契约。

### 6.2 边界与折叠

- 栏间仅显示一条低对比度分隔线，拖拽热区可大于视觉线宽。
- resize hover/drag 时分隔线切换为动作蓝色，并显示轻量反馈。
- 折叠按钮在栏边缘保持可发现但不抢占视觉；键盘 focus 必须清晰。
- 不再叠加“面板边框 + 外层玻璃边框 + 拖拽线”的三重边界。

### 6.3 顶部栏

- 高度控制在 56–64px。
- 左侧返回按钮使用圆形轻按钮，随后为文献本标题和小型状态标签。
- 访客状态用单行紧凑 chip 或说明，不使用覆盖三栏的大型悬浮横幅。
- 宿主登录动作仍可发现，但不遮挡工作台标题、工具或滚动内容。

## 7. 空状态与工具密度

### 7.1 中栏空状态

- 图标、标题、说明整体上移，缩短大面积空白。
- 快捷问题由 8 张大型卡片收敛为更紧凑的 2–4 个建议入口，或采用可横向/换行的轻按钮组。
- 未选择证据时明确说明下一步；按钮禁用必须说明原因，不能点击无反馈。

### 7.2 右侧工具区

- 保留真实能力分组，但卡片高度、图标容器和描述行距统一缩小。
- 同级工具使用一致的点击区域、选中态、loading、不可用和错误反馈。
- 切换工具后在下方明确打开对应工作区；不能让用户误以为点击无效。
- 工具入口无真实能力时必须隐藏或显示明确不可用说明，禁止假按钮。

### 7.3 左侧证据区

- “发现信源”、上传、目录选择和来源列表建立清晰的主次层级。
- 空态只保留一个主要行动和一个次要示例入口。
- 加入文献库、刷新保持、删除清理继续作为非付费回归门禁。

## 8. 动效原则

动效服务于位置、状态和因果，不做背景持续动画：

- 页面/面板进入：200–240ms，轻微透明度与 4–8px 位移。
- 工具选中、卡片 hover、focus：160–200ms。
- 面板折叠/展开/调整：220–260ms，使用标准 ease-out。
- loading 使用局部 skeleton 或状态指示，不让导航图标持续旋转。
- 不使用大幅弹跳、持续发光、视差或影响阅读的循环动画。
- `prefers-reduced-motion: reduce` 下取消位移和非必要过渡，仅保留立即状态切换。

## 9. 状态与错误反馈

- 首帧加载：稳定骨架或空壳，不闪现错误页面或营销页。
- iframe/宿主错误：提供可重试和返回平台的明确路径。
- guest workspace：显示当前会话隔离说明，但不遮挡内容。
- 工具 loading、empty、error、retry 由对应工具区域承担，不用全屏覆盖。
- 键盘 Tab 顺序按左栏、主工作区、右栏；所有交互元素具备可见 focus ring。

## 10. 不变量

以下行为属于发布阻断条件，任何视觉改动均不得回退：

- 每个 guest 浏览器会话使用独立 workspace/account scope。
- 登录用户的 workspace 恢复和账号绑定保持原契约。
- iframe、paper-host `postMessage` bridge、`hideVirtualClassroom` 和返回科教平台路径保持可用。
- 发现信源、文献增删、刷新保持和当前服务端 API 契约不变。
- 高成本登录开关、provider 和计费逻辑不在本设计中修改。

## 11. RED/GREEN 与验收

### 11.1 自动化 RED

实现前必须建立以下真实行为失败证据：

1. 嵌入 `view=notebooks` 的首个可见帧出现营销 CTA 或 KnowTrail 品牌。
2. 嵌入文献本首页/工作台仍出现 KnowTrail 字样或 BrandMark。
3. 独立根路径仍需保留营销页，防止误删。
4. 三栏布局存在重复边界、页面横向溢出或折叠控件不可键盘访问。
5. `prefers-reduced-motion` 下仍存在非必要位移动画。

测试必须挂载实际页面/组件并执行路由或交互，不以源码字符串匹配作为主要证据。

### 11.2 GREEN 验收

- 从科教平台点击智能体后，首个有意义画面直接进入文献本列表，不出现营销落地页。
- 嵌入文献本列表和工作台不出现 KnowTrail 品牌；独立站仍保留品牌。
- 新建文献本、打开文献本、返回列表、刷新保持均可用。
- 1440px 下三栏约为 272px / 弹性 / 420px，无页面级横向溢出。
- 分隔拖拽、折叠、hover、focus 和 reduced-motion 行为正确。
- 中栏空状态与右侧工具卡密度明显收敛，所有可见按钮都有真实反馈。
- 正式域 console error 为 0；失败请求具备可解释反馈和恢复路径。
- guest workspace/account 隔离、postMessage、hideVirtualClassroom 和宿主返回路径通过。
- 390px 仅验证无新增阻断，不以本轮改造手机壳层。

## 12. 发布与回滚

1. 先完成定向行为测试、全量 validate、TypeScript/lint、production build、Linux package、diff/secret/debug scan。
2. 使用独立 standby 端口验证正式构建、健康 JSON、iframe 和关键用户路径。
3. 发布采用时间戳 release、`dirty=false` manifest 和原子 symlink 切换。
4. 发布后复核正式域、PM2、Nginx、平台 health/ready、DB、timer、console 与横向溢出。
5. 任一发布门禁失败，立即切回 previous，并保留失败候选用于审计。

## 13. 完成定义

本轮完成不是“换了颜色”，而是同时满足：嵌入入口首帧正确、品牌边界正确、三栏结构精细、状态反馈完整、动效克制可关闭、核心数据与宿主契约无回退，并在正式域名通过真实用户路径验收。
